Communication entre composants

On parle de composant Enfant quand on utilise un composant dans un autre ( son parent ), via sa balise / son selecteur

Un composant enfant peut lui même être parent d'un composant.

Un composant peut utiliser plusieurs composants enfants, et réutiliser plusieurs fois le même composant.

Communication descendante

Communication du parent vers l'enfant

Pour envoyer des données d'un parent vers un enfant, on utilise les Input

Un composant peut avoir un nombre illimité d'inputs.

Les Inputs peuvent avoir n'importe quel type : primitifs, tableaux, objets, interfaces, ...

Communication ascendante

Communication de l'enfant vers le parent

L'enfant ne passe jamais de données de manières permanente au parent.

La communication de l'enfant vers le parent se passe sous forme d'événement.
